The intellectual property

Word-level annotation — not just verses.

Every word in the NLT corpus carries structured metadata: who speaks, to whom, what role, and which of seven spheres of society it touches.

The universal address

Every dataset joins on the monad.

Emdros assigns each word an integer position in reading order (~1.79M monads). SQLite spans store first and last monad ranges. A word at monad m belongs to span s when s.first ≤ m ≤ s.last.

What works right now

Three access paths — no rebuild required.

A

SQLite direct

Open SourceView.sqlite3 in any SQL client. Full span/actant/sphere analytics work today. Example queries in APP_AND_DATA_HANDOFF.md §10.

B

Python extractor

python3 tools/extract_metadata.py → build/metadata/ with actants.json, spans.json, spans.csv, lookups.json. No Emdros build needed.

C

Corpus harness

sh tools/build_dump_words.sh then run dump_words on SourceView.bpt. Decrypts words + per-word spheres offline. Key committed in repo.

Canonical output shape

One row per word — the future API.

After rebuild, every word becomes a self-contained record. No Emdros or Realm needed at runtime — just lookup tables and a flat file or Postgres.

Keep annotation separate from NLT text for licensing: ship tags freely, swap in any licensed translation via monad alignment.

Metadata gap analysis

What we have — and what we can add.

The current dataset is rich but not exhaustive. The six-tool suite (SourceView, SphereView, VerbView, CommandView…) planned layers that were designed but never shipped into the corpus.

Planned tools · not in current corpus

VerbView & CommandView layers.

VerbView · designed, not shipped

Per-word verb morphology

Every verb tagged by tense, mood, voice, person, number — enabling "show all imperatives God speaks" or "aorist vs imperfect distribution by book." Requires Hebrew/Greek alignment or NLP tagging pipeline on top of monads.

tensemoodvoicestemstrong/weak verb
CommandView · designed, not shipped

Commands & imperatives

Explicit command/imperative classification per word or span — "honor your father," "go and sin no more." Books like Proverbs and Ecclesiastes reference imperatives editorially; CommandView would make them queryable at scale.

imperativejussiveprohibitionblessing formula

Effort: Medium-high — requires original-language resources or ML tagging, then monad-anchored join like existing sphere flags.

Observable metadata · not yet tagged

Location, semantics, and cross-refs.

Geographic tags

Location per word/span

Place names (Jerusalem, Galilee, Egypt) as monad-level or span-level tags — enables "everything spoken in Samaria" or map visualisations.

Semantic type

Word class beyond verb

Noun/verb/adjective, named entity (person, place, group), quotation boundary — standard NLP layers on monads.

Cross-references

Inter-textual links

Typed allusions and parallel passages (OT↔NT) as first-class edges in the knowledge graph.

Original language

Hebrew/Greek lemmas

Strong's, BDB, or morph tags aligned to monads — unlocks VerbView and cross-translation at the root level.

Sentiment / tone

Emotional register

Judgment, comfort, lament, praise — useful for pastoral matching and devotional tools.

Discourse structure

Rhetorical moves

Question, command, promise, prophecy, parable frame — complements role/sphere for Discovery queries.

The web reader ships the GAP Testament layer: OT quotations and intentional allusions tagged to NT verses, with per-book charts at /dataCharts. Nearly 25% of NT verses carry a clear Hebrew Testament reference — tap a word to see linked passages (e.g. GEN 12:7, ISA 61:1).
